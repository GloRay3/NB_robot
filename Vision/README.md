# 视觉与自瞄

> - 背景概要
>
你很不幸的穿越到了美剧**黑袍纠察队**。站在你面前的是**Homelander**。  
![Homelander](../photos/Homelander.jpeg)  
这是本剧中的最大**反派**。  
你正好撞见了他**行凶**。他已经发现了你，并且露出了他标志性的微笑，准备将你**灭口**。  
正在这危急存亡之际，你发现你的身边竟然恰好有一门**粒子激光炮**！它可以对**祖国人**造成行之有效的伤害，保住你的性命。  
又又又非常不幸的是，这门激光炮没有手动控制的接口。你必须编写简单的**自瞄**程序，来控制这门激光炮，以保住你的性命。  
祝你好运！**God Bless You!**
> - 任务配置
>
基于**ubuntu22.04 LTS**,以及**ros2 humble**。部署**yolov8**模型，实现目标检测功能。  
文件里已提供了需要进行检测的**视频源(orivedio.mp4)文件**。您可以使用官方提供的最小预训练检测模型**yolov8n.pt**文件进行模型推理。(参考的yolo版本为**v8.2.0**)。
提供的视频源分辨率为**1280x720**。帧数为**30fps**。在使用模型yolo模型推理时，不指定画面参数时，检测输出的画面大小为**384x640**。无需修改。
提供**ament_camke**类型的功能包，用于接收你所传递的**信息**以及进行**反馈**。
> - 你所需要完成的任务
>
编写一个**ament_python**类型的功能包，使用**话题通信**与提供的功能包进行交互。  
在运行开始时，**ament_cmake**里的节点会**随机**生成一个**炮台的初始位置**，并通过话题**turret_position**发布(发布的坐标为**x,y**两个**int**类型数据,在像素**384x640**范围内)。  
你的**ament_python**类型的功能包**订阅**话题**turret_position**，得到炮台的初始位置。你的功能包里的**程序文件**调用**yolov8**模型推理后，将检测的目标坐标通过**话题通信**，以**x1,y1,x2,y2**四个**int**类型数据发布到话题**target_position**(像素**384*640**内)。并通过结合在订阅话题**turret_position**得到的初始坐标，发布**两位float**线坐标**x11,y22**控制炮台移动(同样发布在话题**target_position**中)。
提供的功能包**ament_camke**会订阅你编写的话题**target_position**。在得到**x11,y22**后，程序内会修正**炮台的位置**(有一定限制)。在规定的周期内，会将**炮台的位置**与接收到的**x1,y1,x2,y2**进行对比，若**炮台的位置**在**x1,y1,x2,y2**范围内，会增加**得分值**。  
**ament_cmake**里节点继续发布**炮台位置**。直到视频源播放结束。  
> - 任务评估
>
依据**得分值**判定结果。得分值到达一定结果算任务完成。
